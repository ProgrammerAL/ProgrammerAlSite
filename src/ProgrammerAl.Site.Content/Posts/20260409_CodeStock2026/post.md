__Session Abstract__: 
Did you know a C# project can detect when there's a known security vulnerability for a NuGet package it's using? Or that you can manage warnings/errors/suggestions for an entire C# project in a single file, without installing an extension?

C# has a handful of free features we can use to enforce high code quality. In this session, we'll look at those C# features and discuss what it takes to use them and create our own C# pit of success.

## What's in a container image?

__Session Abstract__: 
Containers run from image files, and image files are created by Dockerfiles. As developers we make our Dockerfiles start with a base image and copy files into it to make the final image. But why do we need the base image? How do we know what's in it? What do we gain or lose by using that base image vs different ones? Other than the files we add, what else is in the final image?

In this session we'll answer these questions with a mix of slides and demos. Going over the basics of what an image is made of, spend time dissecting one, and even make our own from scratch. By the end of the session, we should have a better understanding of what an image is, what files are in there, and why it exists that way.
